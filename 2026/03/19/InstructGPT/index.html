InstructGPT

1.预训练语言模型（Pretrained LM）

2.监督微调（SFT, Supervised Fine-Tuning）

3.RLHF：奖励模型 + 强化学习优化

一、预训练

预训练阶段，模型的目标通常是：给定前文，预测下一个token

预训练 = 让模型拥有广泛能力

对齐训练 = 让模型学会如何使用这些能力

模型在海量文本上训练，学习：

  • 语法
  • 事实模式
  • 常见问答结构
  • 代码风格
  • 摘要、翻译、解释等任务的隐式模式

这个阶段得到的是一个基础模型

预训练更像是在学“语言世界的分布”，而不是学“在聊天任务里怎样做才最符合用户期待”。

没有额外对齐时，模型可能会：

  • 不按格式回答
  • 明明问的是 A，却展开去讲 B
  • 写得冗长、散漫
  • 把错误说得很自信
  • 对危险请求也照样给出高质量回答
  • 倾向生成“像互联网上常见文本”的东西，而不是“人真正想要的东西”

二、监督微调

SFT 的作用是给模型一个很重要的初始偏向：

  • 学会把“用户指令”当回事
  • 学会常见任务的回答范式
  • 学会礼貌、清晰、结构化表达
  • 学会在部分危险请求上拒绝

所以 SFT 可以理解为：先用人工示范，把模型从“会续写文本”拉到“像助手一样回答”

如何做？

人工标注者会看到一些 prompt，例如：

  • “帮我写一封正式邮件”
  • “用简单的话解释牛顿第一定律”
  • “总结下面这段内容”
  • “这个请求不安全，请拒绝并解释原因”

然后标注者写出一个高质量示范答案。

模型再用这些“输入—理想输出”样本做监督学习，目标是：

  • 给定用户指令，直接模仿高质量回答

局限？

人类偏好很复杂，很难靠单一“标准答案”完全表达出来

比如一个问题的回答可能有很多合理版本：

  • 更简洁
  • 更详细
  • 更安全
  • 更有帮助
  • 更少废话
  • 更符合上下文语气

单纯监督学习往往难以表达这种“相对偏好”

这就引出下一步

三、RLHF

1.训练奖励模型

很多任务不是只有一个正确答案，而是有多个“都可以，但好坏不同”的答案。让人工说“哪个最好、哪个最差”，通常比让人工直接写标准答案更容易。

奖励模型学什么？

奖励模型的输入通常是：

  • prompt
  • 一个候选回答

输出一个分数，表示：

  • 这个回答有多符合人类偏好

于是，原来模糊的“人更喜欢什么”被近似成了一个可计算的打分函数

2.RLHF

这里的问题是：

  • 一个 prompt 可以有很多可接受回答
  • 我们更关心“整体质量”和“偏好排序”
  • 而不是某个固定参考答案

强化学习擅长处理这种情况：

  • 模型生成一个完整回答
  • 奖励模型给它打一个总分
  • 再根据这个分数来更新模型参数

所以这里优化的不是“和标准答案逐 token 一致”，而是：

  • 整段回答最终获得的偏好分数

3.PPO（Proximal Policy Optimization）

PPO 试图让模型朝“更高奖励”的方向更新，但又不能一下改太猛。

为什么不能改太猛？

因为如果模型只顾着刷高奖励，可能会出现：

  • 语言变得怪异
  • 过度迎合奖励模型
  • 丢失原来流畅自然的能力
  • 学会“投机取巧”而不是真正更好地回答

所以 PPO 的一个重要思想是：

  • 更新要谨慎
  • 既要提高奖励，又要别离原模型太远

这就是为什么 RLHF 常常还会加入一个“不要偏离初始模型太多”的约束项
